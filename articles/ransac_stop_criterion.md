# RANSAC停止准则的修正：从近似到精确

## 文章概述

这篇题为"Fixing the RANSAC Stopping Criterion"的论文由Johannes Schönberger、Viktor Larsson和Marc Pollefeys共同撰写，发表于2025年。文章揭示并解决了计算机视觉领域一个长期存在的根本性问题：RANSAC算法停止准则中的近似误差。

## 背景介绍

### RANSAC算法的重要性
RANSAC（Random Sample Consensus）自1981年由Fischler和Bolles提出以来，一直是计算机视觉中最常用的鲁棒估计算法之一。它广泛应用于：
- 运动分割
- 立体视觉
- 几何基元拟合
- 图像拼接
- 多模型拟合
- 相机定位
- 运动恢复结构

### 问题的根源
RANSAC采用"假设-验证"框架，通过随机采样测量值来生成模型假设，然后验证这些假设以选择具有最大一致性的模型。算法的核心在于自适应停止准则，该准则依赖于采样全内点集的概率计算。

## 历史问题：近似停止准则

### 传统近似方法
自原始RANSAC论文以来，几乎所有后续变体和实现都使用以下近似概率：
$$
Pₐ = pᵏ
$$
其中：
- p：观测到的内点比例
- k：最小样本大小

### 精确概率公式
论文指出，正确的精确概率应该是：
$$
Pₑ = ∏(pn-i)/(n-i)  (i从0到k-1)
$$
或者等价表示为：
$$
Pₑ = C(pn, k) / C(n, k)
$$
其中：
- n：总测量数
- pn：内点总数

## 数学分析与误差影响

### 近似误差的理论分析
论文通过理论推导证明：
$$
Pₑ < Pₐ
$$
这意味着传统近似总是高估了真实概率，导致RANSAC过早停止。

### 误差影响因素
近似误差在以下情况下尤为严重：
1. **测量数量少**：当n较小时，近似误差显著
2. **内点比例低**：p值越小，误差越大
3. **模型复杂度高**：k值越大，误差越明显

- **欠采样**：RANSAC在挑战性场景中过早停止
- **模型失败**：无法找到好的模型
- **性能下降**：在低内点比例场景中表现不佳

## 解决方案与实现

### 精确停止准则的实现
实现精确概率计算非常简单：

```python
def exact_all_inlier_probability(n, p, k):
    """计算精确的全内点采样概率"""
    pn = int(p * n)  # 内点总数
    if pn < k:
        return 0.0
    
    probability = 1.0
    for i in range(k):
        probability *= (pn - i) / (n - i)
    return probability
```

### 停止准则计算
```python
def compute_iterations(s, P):
    """计算所需的迭代次数"""
    if P >= 1.0:
        return 1
    return math.ceil(math.log(1 - s) / math.log(1 - P))
```

## 结论与意义

### 主要贡献
1. **理论分析**：首次系统分析了RANSAC停止准则中的近似误差
2. **精确解决方案**：提出了简单有效的精确概率计算方法
3. **实验验证**：在多个计算机视觉问题上验证了改进效果

### 实际意义
- 在挑战性场景中显著提高模型估计成功率
- 减少灾难性估计失败
- 为计算机视觉社区提供重要的算法修正

### 局限性
- 精确方法会增加运行时间，特别是在困难场景中
- 未考虑测量噪声、最小求解器稳定性等其他因素
- 未在所有RANSAC变体上进行全面测试
